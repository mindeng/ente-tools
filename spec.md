根据对 ../ente 项目的分析，得出 ente 对文件 hash 的计算方式如 ./ente-file-hash.md 文档中所述。

参考上述资料，请用 golang 开发一个命令行工具，要求如下：

1. 按照 ente 的方式，计算指定路径下所有受支持文件类型的文件的 hash，请注意 Live Photo 
   的识别和特殊处理（参考 ente 的识别和处理逻辑），并支持目录的递归遍历。如有重复文件需在报告中输出；
2. 将计算的 hash 存入指定的数据库文件中，可采用 bbolt 数据库来存储，注意针对 Live Photo 的处理；
3. 一个目录对应一个 db 文件。注意在 db 中，文件路径仅保存相对于指定顶层目录的相对路径，不存绝对路径，
   避免顶层路径发生变化时，路径失效的问题；
4. 可对两个目录下的文件 hash 集合进行对比，输出差异报告，例如 A 目录和 B 目录相比，有多少文件一致，
   缺失了哪些文件， 多出了哪些文件。每次比较前，先遍历目录 ，并根据情况判断是否需要计算或更新文件 hash 
   并写入数据库（根据文件大小、最后修改时间等元数据是否发生变化进行判断）。 

请先进行方案设计，确认方案无误后，再进行开发、测试工作。
